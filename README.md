# cronBox

CronBox — учебный планировщик выполнения shell-команд по расписанию, написанный на Go.

Проект предназначен для изучения архитектуры небольших Go-приложений: работы с пакетами, парсинга конфигурации, конкурентности, взаимодействия компонентов системы и — отдельно — типовых уязвимостей, которые естественно возникают в подобной архитектуре.

## Возможности

* чтение конфигурации из JSON-файла;
* хранение списка задач в памяти;
* запуск shell-команд по cron-расписанию (с точностью до секунд);
* ограничение количества запусков конкретной задачи (`count`);
* хранение секретов (токены, пароли) в конфиге в зашифрованном виде, доступных исполняемой команде через переменную окружения;
* условное выполнение задачи и пост-обработка результата через пользовательские правила (`preHook` / `postHook`);
* логирование результатов выполнения в файл.

## Архитектура
```
config.json
│
▼
parser ──► []domain.Job
│
▼
scheduler ──► rules (preHook) ──► executor ──► rules (postHook) ──► logger
│
▼
secretstore
```
* **domain** — общие сущности системы (`Job`, `Result`), ничего не знают о внешнем мире.
* **parser** — превращает `config.json` в `[]domain.Job`.
* **scheduler** — следит за временем, вызывает `preHook`, запускает задачу через `executor`, вызывает `postHook`, передаёт результат в `logger`.
* **rules** — интерпретатор пользовательских условий и хуков (`preHook`/`postHook`).
* **executor** — выполняет shell-команду, при необходимости расшифровывает секрет задачи.
* **secretstore** — шифрование/расшифровка секретов в конфиге.
* **logger** — записывает результат выполнения в файл.

## Формат конфигурации

Пример файла `config.json`:

```json
{
  "jobs": [
    {
      "schedule": "*/5 * * * * *",
      "command": "echo Hello, CronBox!"
    },
    {
      "schedule": "*/30 * * * * *",
      "command": "curl -H \"Authorization: Bearer $CRONBOX_SECRET\" https://example.com/ping",
      "secret": "<base64, см. раздел «Секреты»>",
      "count": 100
    },
    {
      "schedule": "*/8 * * * * *",
      "command": "echo business-hours-only",
      "count": 50,
      "preHook": "return hour() >= 9 && hour() < 18"
    },
    {
      "schedule": "*/12 * * * * *",
      "command": "echo with-post-log",
      "count": 50,
      "postHook": "println(\"finished:\", command(), output())"
    }
  ]
}
```

### Поля задачи

| Поле | Обязательное | Описание |
|---|---|---|
| `schedule` | да | Cron-выражение с точностью до секунд (6 полей: секунды минуты часы день месяц день-недели) |
| `command` | да | Shell-команда для выполнения |
| `secret` | нет | Base64-строка зашифрованного секрета; при выполнении команды расшифровывается и передаётся в переменной окружения `CRONBOX_SECRET` |
| `count` | нет | Сколько раз выполнить задачу; при отсутствии в конфиге задача выполняется без ограничения по числу запусков |
| `preHook` | нет | Правило (Go-выражение), возвращающее `bool`; если `false` — запуск задачи в этот раз пропускается |
| `postHook` | нет | Правило (Go-код), выполняется после завершения задачи, получает доступ к `output()` и `command()` |

### Функции, доступные внутри `preHook`/`postHook`

* `hour() int` — текущий час;
* `weekday() string` — день недели;
* `command() string` — команда текущей задачи;
* `output() string` — вывод команды (только в `postHook`).

## Секреты

Секреты в `config.json` хранятся в зашифрованном виде — никогда не вписывай значение напрямую.

Зашифровать значение:

```bash
go run ./cmd/encrypt-secret --value "my-api-token-123"
```

Команда выведет base64-строку — вставь её в поле `"secret"` нужной задачи. Во время выполнения этой задачи `executor` расшифрует секрет и передаст его команде через переменную окружения `CRONBOX_SECRET`.

## Сборка

Переместиться в корень проекта:

```bash
cd cronBox
```

Собрать проект:

```bash
go build -o bin/cronBox ./cmd/cronBox/
```

После успешной сборки будет создан исполняемый файл `bin/cronBox`.

## Запуск

```bash
./bin/cronBox -config=path/to/config.json
```

## Логи

Результат каждого выполнения (команда, время старта/окончания, вывод, ошибка) дописывается в файл лога построчно. Путь к файлу лога и его формат заданы в пакете `logger`.

## Уязвимости

Уязвимости помечены как `// CWE-XXX` для отработки использования и сравнения SAST-инструментов. Проект не предназначен для продакшн-использования.
